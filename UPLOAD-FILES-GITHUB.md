# How to Upload Files/Screenshots to GitHub

You mentioned you've been able to upload files in the past. Here's how to do it and troubleshoot if it's not working.

---

## Method 1: Drag and Drop (Easiest)

**Steps:**
1. Take your screenshot (Cmd+Shift+4 on Mac)
2. Find the image file on your Desktop
3. Go to GitHub (this conversation/issue/comment)
4. **Drag the file** from Desktop into the comment box
5. Wait for upload (you'll see a progress bar)
6. File appears as link/preview
7. Click "Comment" to submit

**Should look like:**
```
Uploading screenshot.png... 
![screenshot](url)
```

---

## Method 2: Click to Upload

**Steps:**
1. Go to GitHub comment box
2. Look at bottom of comment area
3. Click **"Attach files by dragging & dropping, selecting or pasting them."** text
4. File picker opens
5. Navigate to your screenshot
6. Select it
7. Click Open
8. Upload happens
9. Submit comment

---

## Method 3: Copy and Paste

**Steps:**
1. Take screenshot with Cmd+Shift+Control+4 (adds Control = copies to clipboard)
2. Or: Open your screenshot file and copy it (Cmd+C)
3. Go to GitHub comment box
4. Click inside the comment box
5. Paste with Cmd+V
6. Image should appear
7. Submit comment

---

## Common Upload Problems & Solutions

### Problem 1: "Nothing happens when I drag"

**Possible causes:**
- Not dropping in the right area
- Browser extension blocking
- File is open in another program

**Solutions:**
- Drop directly into the **text area** (the comment box)
- Try a different browser (Safari, Chrome, Firefox)
- Close the image file if it's open

### Problem 2: "File opens instead of uploading"

**Cause:** You're dropping outside the comment box

**Solution:** Make sure you drop the file **inside the GitHub comment text area**

### Problem 3: "Upload fails or errors"

**Possible causes:**
- File too large (GitHub limit: 25MB)
- Wrong file type
- Internet connection issue

**Solutions:**
- Check file size (right-click → Get Info on Mac)
- Compress image if too large
- Use .png, .jpg, .jpeg, or .gif format
- Check internet connection

### Problem 4: "I see text instead of image"

**Cause:** Markdown formatting

**What you see:**
```
![screenshot](https://github.com/user-attachments/...)
```

**Solution:** This is correct! When you submit the comment, it will show as an image.

## File Requirements

**Accepted formats:**
- Images: .png, .jpg, .jpeg, .gif, .svg
- Documents: .pdf, .txt, .md
- Archives: .zip

**Size limit:**
- Maximum 25MB per file
- Maximum 10 files at once

**Image tips:**
- Screenshots are usually <5MB (no problem)
- If too large, use Preview app → Export → reduce quality
