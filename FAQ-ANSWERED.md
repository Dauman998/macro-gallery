## 🖥️ LOCAL PREVIEW (See Changes Instantly)

### Method 1: Manual Preview (Easy!)

**Steps:**
1. Edit `index.html` in TextEdit/VS Code
2. Save the file (Cmd+S)
3. Go to Finder
4. Double-click `index.html`
5. Opens in browser → **You see your site!**
6. Make more changes
7. Save again
8. Switch to browser
9. Press **Cmd+R** to refresh
10. **See your new changes!**

**This is LOCAL only** (just on your Mac, not on internet yet)

### Method 2: Live Server (Auto-Refresh!)

**Best way - Changes show automatically!**

**Setup (One-time, 2 minutes):**
1. Download VS Code: https://code.visualstudio.com
2. Install it
3. Open VS Code
4. Click Extensions icon (4 squares in left sidebar)
5. Search "Live Server"
6. Install the one by Ritwick Dey
7. Done!

**Usage (Every time):**
1. Open VS Code
2. File → Open Folder → Select `macro-gallery`
3. Click `index.html` in left sidebar
4. Right-click → "Open with Live Server"
5. **Browser opens automatically!**
6. Edit your file
7. Press Cmd+S to save
8. **Browser refreshes automatically!**
9. See changes instantly! ✨

**No more manual refresh needed!**

### What You See in Preview

The preview shows **exactly** what your site will look like:
- Your header
- Your photos
- Hover effects
- Mobile responsive design
- Everything!

**It's the REAL site**, just running on your Mac instead of the internet.

## 🎯 Best Practices

### Work Locally First
✅ **DO:** Make changes locally, preview, test  
✅ **DO:** Deploy when you're happy  
❌ **DON'T:** Edit directly on GitHub (no preview!)  
❌ **DON'T:** Deploy every tiny change (batch them)  

### Save Often
- Edit a bit → Cmd+S → Preview
- Edit more → Cmd+S → Preview
- Get in this rhythm!

### Use Browser Refresh
- Made changes?
- Cmd+R in browser
- See them appear!

### Check Before Deploy
Before uploading to GitHub:
- [ ] Preview locally - does it look good?
- [ ] Test hover effects
- [ ] Check mobile view (Cmd+Option+I in browser)
- [ ] All photos load?
- [ ] Titles correct?
- [ ] Ready? Deploy!
